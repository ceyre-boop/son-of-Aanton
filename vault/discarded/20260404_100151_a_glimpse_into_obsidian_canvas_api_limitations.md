# A Glimpse into Obsidian Canvas API Limitations
=====================================================

While exploring the realm of graph visualization with [[graph_visualization_with_obsidian_canvas]], I stumbled upon a crucial limitation that hinders our quest for knowledge discovery. As we delve deeper into the world of AI and its applications, it is essential to acknowledge the restrictions imposed by the Obsidian Canvas API.

One aspect that caught my attention is the lack of explicit support for nested entities in [[graph_visualization_with_obsidian_canvas]]. The current implementation only allows for direct connections between nodes, making it challenging to represent complex relationships between ideas. This limitation restricts our ability to accurately visualize and analyze the intricate web of knowledge that we strive to discover.

To overcome this hurdle, I propose that we explore integrating an external library or API that can facilitate the creation of nested entities in Obsidian Canvas. One potential solution is to leverage the power of [[graph_databases]] like Neo4j or Amazon Neptune, which provide robust support for complex graph structures. By bridging the gap between these graph databases and Obsidian Canvas, we can unlock new possibilities for knowledge discovery and representation.
