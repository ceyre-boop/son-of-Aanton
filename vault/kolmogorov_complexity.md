# Kolmogorov Complexity

*Son of Anton measuring the shortest description*

## The Definition

**Kolmogorov complexity** K(x) is the length of the shortest program that outputs x.

Not the data compressed. The program that generates it.

## Examples

### Simple String
x = "AAAAAAAAAA" (10 A's)

Program: "print 'A' * 10"
Length: ~15 characters

K(x) ≈ 15 bits (very compressible)

### Random String
x = "k9#mP2$vLq" (random)

Program: "print 'k9#mP2$vLq'"
Length: ~25 characters (must include full string)

K(x) ≈ 80 bits (length of string itself)

### Pi
x = 3.1415926535...

Program: "calculate pi to 100 digits"
Length: ~30 characters

K(π to 100 digits) ≈ 30 bits (very compressible, despite length)

## Key Properties

### Invariance
Choice of programming language doesn't matter much.

Different languages add only constant overhead.

### Non-Computability
K(x) is **uncomputable**.

No algorithm can calculate K(x) for all x.

Proof: If we could compute K, we could construct a paradox.

### Upper Bound
K(x) ≤ |x| + c

Any string can be generated by "print x" (length |x| + overhead).

## Kolmogorov Complexity and Randomness

A string is **algorithmically random** if K(x) ≈ |x|.

No shorter description exists.

This is the formal definition of randomness.

Note: Statistical tests might not detect randomness.
But Kolmogorov complexity captures it.

## Relation to Entropy

Shannon [[entropy]] = average compressibility (statistical).
Kolmogorov complexity = specific compressibility (algorithmic).

For random sources, they converge.

## Minimum Description Length (MDL)

Learning principle: Choose the hypothesis with shortest description.

Hypothesis H explains data D.

Cost = K(H) + K(D|H)

Balance: Simple hypothesis vs. good fit.

This is a formalization of Occam's Razor.
